<Note>
  All UnPay payout requests require the payload to be encrypted using **AES-256-CBC** with output in **HEX format**. Your AES Key and IV are generated from the [UnPay Dashboard](https://app.unpay.in).
</Note>

## Overview

UnPay encrypts payout data end-to-end to protect sensitive financial information in transit. Before calling any payout endpoint, you must:

1. Build the **plaintext JSON payload** with your payout fields
2. Encrypt it using **AES-256-CBC** with your dashboard-issued AES Key and IV
3. Encode the encrypted bytes as a **HEX string** (not Base64)
4. Send the HEX string as the `body` field in your API request

## Get your AES Key and IV

Log in to [https://app.unpay.in](https://app.unpay.in) → **Settings** → **API Security** to generate your:

- **AES Key** — secret key
- **AES IV** — initialization vector

<Warning>
  Keep your AES Key and IV private. Treat them like passwords — never expose them in client-side code or public repositories.
</Warning>

## Plaintext payload structure

### Option A — Existing beneficiary

Use this when the beneficiary is already registered in your UnPay account.

```json
{
  "debit_account_id": "[UNPAY_ACCOUNT_ID]",
  "bene_id": "BENE123456789",
  "bene_bank_id": 1,
  "amount": [AMOUNT],
  "payment_mode": "[PAYMENT_MODE]",
  "narration": "[NARRATION_TEXT]",
  "purpose": "[PURPOSE_TEXT]",
  "notes": "[NOTES_TEXT]"
}
```

### Option B — Inline beneficiary

Use this to send a payout without pre-registering a beneficiary.

```json
{
  "debit_account_id": "[UNPAY_ACCOUNT_ID]",
  "name": "[NAME]",
  "mobile": "[MOBILE_NUMBER]",
  "email": "[EMAIL_ADDRESS]",
  "bank_id": "[BANK_ID]",
  "bank_acc_num": "[BANK_ACCOUNT_NUMBER]",
  "ifsc_code": "[IFSC_CODE]",
  "bank_owner_name": "[BANK_OWNER_NAME]",
  "amount": [AMOUNT],
  "payment_mode": "[PAYMENT_MODE]",
  "narration": "[NARRATION_TEXT]",
  "purpose": "[PURPOSE_TEXT]",
  "notes": "[NOTES_TEXT]"
}
```

## Encryption steps

<Steps>
  <Step title="Serialize the payload to a JSON string">
    Convert your plaintext object to a compact JSON string. Do not add extra whitespace.
  </Step>
  <Step title="Encrypt with AES-256-CBC">
    Use AES-256-CBC mode with your AES Key and AES IV. The output is raw bytes.
  </Step>
  <Step title="Encode output as HEX">
    Convert the encrypted bytes to a HEX string. **Do not use Base64.**
  </Step>
  <Step title="Send in the body field">
    Place the HEX string as the value of the `body` field in your request JSON.
  </Step>
</Steps>

## Code examples

<CodeGroup>

```javascript Node.js
const crypto = require('crypto');

const AES_KEY = Buffer.from('your_32_byte_hex_key_here', 'hex');
const AES_IV  = Buffer.from('your_16_byte_hex_iv_here', 'hex');

function encryptPayload(payload) {
  const plaintext = JSON.stringify(payload);
  const cipher = crypto.createCipheriv('aes-256-cbc', AES_KEY, AES_IV);
  const encrypted = Buffer.concat([
    cipher.update(plaintext, 'utf8'),
    cipher.final()
  ]);
  return encrypted.toString('hex'); // HEX output
}

// Option A — existing bene
const payload = {
  debit_account_id: 'UNPW4065468842',
  bene_id: 'BENE0407717739',
  bene_bank_id: 86,
  amount: 5,
  payment_mode: 'IMPS',
  narration: 'payout',
  purpose: 'test',
  notes: 'test'
};

const encryptedBody = encryptPayload(payload);

// API call
const response = await fetch('https://app.unpay.in/api/v1/tech/product/payouts/pay', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
    'x-api-key': 'your_api_key_here'
  },
  body: JSON.stringify({ body: encryptedBody })
});
```

```python Python
import json
import binascii
from Crypto.Cipher import AES
from Crypto.Util.Padding import pad

AES_KEY = bytes.fromhex('your_32_byte_hex_key_here')
AES_IV  = bytes.fromhex('your_16_byte_hex_iv_here')

def encrypt_payload(payload: dict) -> str:
    plaintext = json.dumps(payload, separators=(',', ':')).encode('utf-8')
    cipher = AES.new(AES_KEY, AES.MODE_CBC, AES_IV)
    encrypted = cipher.encrypt(pad(plaintext, AES.block_size))
    return binascii.hexlify(encrypted).decode('utf-8')  # HEX output

# Option A — existing bene
payload = {
    "debit_account_id": "UNPW4065468842",
    "bene_id": "BENE0407717739",
    "bene_bank_id": 86,
    "amount": 5,
    "payment_mode": "IMPS",
    "narration": "payout",
    "purpose": "test",
    "notes": "test"
}

encrypted_body = encrypt_payload(payload)

import requests
response = requests.post(
    'https://app.unpay.in/api/v1/tech/product/payouts/pay',
    headers={
        'Content-Type': 'application/json',
        'x-api-key': 'your_api_key_here'
    },
    json={"body": encrypted_body}
)
```

```php PHP
<?php
$aesKey = hex2bin('your_32_byte_hex_key_here');
$aesIv  = hex2bin('your_16_byte_hex_iv_here');

function encryptPayload(array $payload, string $key, string $iv): string {
    $plaintext = json_encode($payload, JSON_UNESCAPED_UNICODE);
    $encrypted = openssl_encrypt($plaintext, 'aes-256-cbc', $key, OPENSSL_RAW_DATA, $iv);
    return bin2hex($encrypted); // HEX output
}

// Option A — existing bene
$payload = [
    'debit_account_id' => 'UNPW4065468842',
    'bene_id'          => 'BENE0407717739',
    'bene_bank_id'     => 86,
    'amount'           => 5,
    'payment_mode'     => 'IMPS',
    'narration'        => 'payout',
    'purpose'          => 'test',
    'notes'            => 'test',
];

$encryptedBody = encryptPayload($payload, $aesKey, $aesIv);

$ch = curl_init('https://app.unpay.in/api/v1/tech/product/payouts/pay');
curl_setopt_array($ch, [
    CURLOPT_POST           => true,
    CURLOPT_RETURNTRANSFER => true,
    CURLOPT_HTTPHEADER     => [
        'Content-Type: application/json',
        'x-api-key: your_api_key_here',
    ],
    CURLOPT_POSTFIELDS => json_encode(['body' => $encryptedBody]),
]);
$response = curl_exec($ch);
```

## Field reference

### Option A fields (existing beneficiary)

| Field | Type | Required | Description |
|---|---|---|---|
| `debit_account_id` | string | ✅ | Your UnPay wallet/account ID |
| `bene_id` | string | ✅ | Pre-registered beneficiary ID |
| `bene_bank_id` | integer | ✅ | Internal bank ID of the beneficiary's bank |
| `amount` | number | ✅ | Amount in INR |
| `payment_mode` | string | ✅ | `IMPS`, `NEFT`, or `RTGS` |
| `narration` | string | ✅ | Transaction narration/reference |
| `purpose` | string | ✅ | Purpose of payout |
| `notes` | string | ❌ | Internal notes |
| `otp` | string | ✅ (user route only) | OTP for dashboard-initiated payouts |

| `bank_acc_num` | string | ✅ | Beneficiary bank account number |
| `ifsc_code` | string | ✅ | Bank IFSC code |
| `bank_owner_name` | string | ✅ | Name on the bank account |
| `amount` | number | ✅ | Amount in INR |
